# 📝 Moss Arcade - Game Design Document (기획서)

## 1. 게임 개요 (Game Overview)
- **게임명**: Moss Arcade
- **장르**: Web-based Casual Arcade / PvP Betting Platform (모스랜드 메타버스 기반)
- **플랫폼**: Web (PC/Mobile Responsive)
- **타겟 유저**: 가볍게 즐길 수 있는 미니게임과 경쟁을 좋아하는 유저
- **핵심 키워드**: 실시간 대전, MOC/MMOC 듀얼 토큰, PvP 배팅, 등급 경쟁, UGC 마켓플레이스

## 2. 핵심 게임 루프 (Core Game Loop)
1. **로비 진입**: 유저는 로비에서 게임을 선택하고 현재 진행 중인 매치를 확인합니다.
2. **매칭/대기**: 게임 룸에 입장하여 다른 플레이어를 기다리거나 관전합니다.
3. **플레이/배팅**: 직접 게임에 참여하며 자신의 **MMOC(인게임 포인트)**를 걸고 대결하거나, 관전하며 승자를 예측 배팅합니다.
4. **결과 및 보상**: 게임 결과에 따라 상대방이 건 MMOC를 획득합니다 (파산 시 Faucet 무료 충전 제공).
5. **성장/수집**: 업적 달성, 일일 퀘스트 완료, 등급 상승으로 콘텐츠 해금. 획득한 MMOC는 언제든 MOC로 출금 가능.

## 3. 유저 플로우 (User Flow)
```
[첫 방문] → 메타마스크 지갑 연결 → 온보딩 튜토리얼
    → [입금소] MOC → MMOC 전환 (1:1)
    → [로비 (Home)]
        ├─ 게임 선택 → 배팅(MMOC) 설정 → 매칭 대기 → 인게임 → 결과 화면 → 로비 복귀
        ├─ 관전 모드 → 승자 예측 배팅(MMOC) → 결과
        ├─ 친구 목록 / 채팅 / 파티 초대
        ├─ UGC 마켓플레이스 / 아바타 꾸미기
        ├─ 리더보드 / 전적 조회
        ├─ 일일 퀘스트 / 업적 확인
        ├─ 토너먼트 참가
        └─ [출금소] MMOC → MOC 전환 → 메타마스크 지갑으로 출금
```

## 4. 온보딩 & 튜토리얼 (Onboarding)
- **Step 1**: 메타마스크 지갑 연결 + 닉네임 설정 + 아바타 기본 선택
- **Step 2**: 입금소 안내 (MOC → MMOC 전환 방법)
- **Step 3**: "첫 게임 해보기" - AI 봇 상대 Neon Deathmatch 1판 체험 (Faucet 포인트 사용)
- **Step 4**: PvP 배팅 방법 안내 (MMOC 걸기 → 대전 → 승자 획득 → 출금 흐름)
- **Step 5**: Faucet(무료 충전소) 위치 안내
- **스킵 가능**: 모든 단계에서 "건너뛰기" 버튼 제공

## 5. 주요 기능 (Key Features)

### 5.1. 로비 (Lobby) - **Poki Style UI**
- **레이아웃 컨셉**: "Discover & Play" - **Poki 스타일 Masonry Grid** (불규칙 크기 카드 그리드)
- **상단 네비게이션 (Top Bar)**:
    - **Logo**: 홈으로 이동
    - **Search Bar**: 게임, 유저, 친구 검색 (자동완성 지원)
    - **User Asset**: 보유 MMOC 잔고 표시 (클릭시 입금소/출금소 이동), 알림(Notification) 아이콘
    - **Profile Menu**: 내 아바타, 설정, 로그아웃 드롭다운

- **메인화면 구성 (Home/Discover) — Masonry Grid**:
    - **레이아웃**: 카로셀 없이, **다양한 크기의 카드를 타일처럼 빈틈없이 배치** (Poki 참고)
    - **카드 크기**: Large(2×2), Medium(1×2 또는 2×1), Small(1×1) 혼합 사용
    - **카드 종류** (게임 외 이벤트/공지/상점도 카드로 혼합 배치):

    | 카드 유형 | 내용 | 크기 예시 |
    |-----------|------|-----------|
    | 🎮 **게임 카드** | 썸네일 + 게임명 + CCU + 호버 시 플레이 버튼 | Large / Medium / Small |
    | 📢 **공지 카드** | 운영 공지, 업데이트 소식, 점검 안내 | Large / Medium |
    | 🎉 **이벤트 카드** | 기간 한정 이벤트, 토너먼트 알림, 시즌 배틀패스 | Large / Medium |
    | 🛒 **상점 카드** | 신규 스킨, 추천 UGC, 기프티콘 교환 프로모션 | Medium / Small |
    | 🏆 **랭킹 카드** | 금주 리더보드 TOP3, 인기 게임 순위 | Small |
    | 👥 **소셜 카드** | 접속 중인 친구, 파티 초대 알림 | Small |

    - **카드 공통 UI**:
        - 호버 시 살짝 확대(scale 1.03) + 네온 글로우 보더
        - 카드 좌상단: 유형 태그 뱃지 (🎮 게임 / 📢 공지 / 🎉 이벤트 / 🛒 상점)
        - 게임 카드: 현재 접속자 수(CCU), 좋아요 비율(%) 표시
        - 이벤트/상점 카드: 남은 기간 카운트다운 뱃지
    - **카드 정렬 우선순위**: 이벤트(핀) → 공지(핀) → 최근 플레이 게임 → 추천 게임 → 인기 게임 → 상점 → 랭킹 → 소셜

- **사이드바 (Sidebar/Drawer)**:
    - **Home**: 메인 로비
    - **Wallet**: 입금소(MOC→MMOC) / 출금소(MMOC→MOC) / 잔고 확인
    - **Avatar**: 내 캐릭터 꾸미기 (마켓플레이스 연동)
    - **Friends**: 친구 목록, 현재 접속 중인 친구 (Join 기능)
    - **Inventory**: 보유 아이템/스킨 확인
    - **Marketplace**: UGC 스킨 거래 + 기프티콘 교환소
    - **Leaderboard**: 실시간 랭킹 (MMOC 보유량, 승률)
    - **Tournaments**: 토너먼트 목록 및 참가

- **소셜 패널 (Right Panel/Chat)**:
    - 친구 목록 및 상태 표시 (접속중, 게임중 등)
    - 1:1 채팅 및 파티 초대 기능

- **통화 체계**:
    - **MOC (Mainnet)**: 유저의 메타마스크 지갑에 보유한 외부 자산 (ERC-20 토큰)
    - **MMOC (In-game)**: 아케이드 내부 전용 포인트. 입금소를 통해 MOC를 입금하면 1:1 즉시 생성
    - **Faucet (무료 충전)**: MMOC 잔고 1,000 미만 시 봇 인증 후 1,000 MMOC 무료 충전 (일 3회 제한)

### 5.2. 인게임 (In-Game) - 공통
- **사이드바**: 로비 메뉴 접근 및 빠른 이동
- **게임 모드 구분**:
    - **프리 모드 (무료)**: 배팅 없이 순수 연습/재미 목적 플레이 (AI 봇 상대 가능). EXP만 획득.
    - **배팅 모드 (유료)**: MMOC를 걸고 PvP 대결. 승자가 판돈 획득.
- **PvP 배팅 패널** (배팅 모드 전용):
    - 양쪽 플레이어가 건 MMOC 표시 (예: Player A 100 MMOC vs Player B 100 MMOC)
    - 배팅 금액 조절 (10, 50, 100, 500 MMOC 단위)
    - 양측 합의 후 게임 시작 (Ready/Playing/Ended)
    - **승자 총취 (Winner Takes All)**: 승리한 플레이어가 양쪽이 건 총 MMOC 획득
- **AI 봇 정책**: 봇은 프리 모드에서만 사용 가능. 배팅 모드에서는 실제 유저만 참여 (봇을 이용한 어빌징 방지)
- **게임 컨테이너**: 실제 게임이 렌더링되는 영역 (Canvas/WebGL)
- **채팅**: 해당 게임 룸 참여자 간 대화

### 5.3. 등급 시스템 (Rank System)
- **개요**: 누적 배팅액 + 승점에 따라 등급이 결정됩니다.
- **등급 체계** (유저 서비스 수준, 저액 기준):
    1. **평민 (Beginner)**: 누적 배팅 0 ~ 5,000 MMOC (배팅 한도: 100)
    2. **중수 (Regular)**: 누적 배팅 5,000 ~ 50,000 MMOC (배팅 한도: 500)
    3. **고수 (Skilled)**: 누적 배팅 50,000 ~ 200,000 MMOC (배팅 한도: 2,000)
    4. **지존 (Master)**: 누적 배팅 200,000 ~ 1,000,000 MMOC (배팅 한도: 10,000)
    5. **도신 (Legend)**: 누적 배팅 1,000,000+ MMOC (배팅 한도: **무제한**)
- **등급 혜택**: 등급이 오를수록 입장 가능한 채널(고액방)이 해금되고, 프리미엄 스킨/칭호가 잠금 해제됩니다.

### 5.4. 매칭 시스템 (Matchmaking)
- **빠른 매칭**: 등급 ±1 범위 내 유저와 자동 매칭 (5초 타임아웃 → 범위 확장)
- **방 만들기**: 커스텀 룸 생성 (비밀번호 설정, 최대 인원 조절, **판돈 설정**)
- **친구 초대**: 파티원과 함께 매칭 큐 진입
- **AI 봇 채우기**: 매칭 실패 시 빈 슬롯에 AI 봇 자동 배치 (프리 모드에서만. 배팅 모드는 실제 유저만 매칭)

### 5.5. 관전 모드 (Spectator Mode)
- **실시간 관전**: 진행 중인 게임방에 관전자로 입장 가능
- **관전 중 배팅**: 관전하면서 실시간으로 승자 예측 배팅 (관전자끼리의 사이드 배팅)
- **딜레이**: 관전 화면 3초 딜레이 (부정행위 방지)
- **관전자 채팅**: 관전자끼리만 보이는 별도 채팅 채널
- **관전자 수 표시**: 게임 룸에 현재 관전자 수 표시 (인기도 지표)

### 5.6. 일일 퀘스트 & 업적 (Daily Quest & Achievement)
#### 일일 퀘스트 (매일 자정 갱신)
| 퀘스트 | 조건 | 보상 |
|--------|------|------|
| 첫 접속 | 로그인 | 경험치 +50 |
| 도전자 | 게임 3판 플레이 | 경험치 +100 |
| 승리자 | 게임 1판 승리 | 경험치 +200 |
| 예측가 | 관전 배팅 3회 | 경험치 +50 |
| 사교왕 | 채팅 메시지 10개 전송 | 경험치 +30 |

- **보상 방식**: 퀘스트 보상은 **경험치(EXP)**와 **칭호/스킨** 등 비재화 아이템으로 지급 (모스코인 직접 지급 없음)
- **퀘스트 리롤**: 1일 1회 마음에 안 드는 퀘스트 변경 가능

#### 업적 시스템 (Achievement)
- **티어 구조**: 🥉 Bronze → 🥈 Silver → 🥇 Gold
- 예시: "첫 승리", "100연승", "10만 MMOC 달성", "도신 등극", "500판 플레이"
- **보상**: 칭호(Title), 아바타 프레임, 한정 스킨 (직접 MMOC 지급 없음)
- **프로필 배지 표시**: 달성한 업적 중 최대 3개를 프로필에 장착 가능

### 5.7. 알림 시스템 (Notification)
- **인앱 알림 (Toast)**: MMOC 변동, 친구 접속, 매칭 완료, 퀘스트 달성
- **알림 센터 (벨 아이콘)**: 읽지 않은 알림 모아보기 (빨간 뱃지 카운트)
- **푸시 알림 (추후)**: 브라우저 푸시로 토너먼트 시작, 친구 초대 알림

## 6. 포함된 게임 (Included Games)
### 6.1. Neon Deathmatch (ID: 8)
- **방식**: 8인 배틀로얄
- **조작**: WASD (이동), SPACE (대시)
- **규칙**: 원형 경기장에서 상대를 밀쳐내어 떨어뜨리는 최후의 1인 승리
- **특징**: 시간이 지날수록 경기장이 좁아짐 (배틀그라운드 자기장 시스템)
- **PvP 배팅 (분배형)**: 8명 × 50 MMOC = 400 MMOC
    - 1등: 240 MMOC (60%)
    - 2등: 100 MMOC (25%)
    - 3등: 60 MMOC (15%)
    - 4~8등: 전액 소멸
- **관전 배팅 옵션**: 1등 예측, 톱3 예측, 최후의 2인 예측

### 6.2. Billiards (ID: 1)
- **방식**: 1v1 당구
- **PvP 배팅**: 두 플레이어가 동일 금액을 걸고 대결, 승자가 총 판돈 획득 (예: 100 MMOC vs 100 MMOC)
- **상태**: 기본 구현 완료

### 6.3. (추가 예정 후보)
- 알까기, 러시안룰렛, 가위바위보, 체스, 하키 퍽

## 7. 토너먼트 시스템 (Tournament)
- **형식**: 8강/16강 **싱글 엘리미네이션** (패배 즉시 탈락)
- **참가비**: 등급별 참가비 (예: 평민방 50 MMOC, 고수방 500 MMOC)
- **상금 풀**: 참가비 총합의 90% (10%는 운영 수수료 = Sink)
- **일정**: 매일 정해진 시간(예: 20:00, 22:00)에 자동 개최
- **대진표 UI**: 실시간 업데이트되는 토너먼트 브래킷 표시
- **관전 연동**: 결승전 자동 관전 방 개설 (로비 배너 홍보)

## 8. 디자인 컨셉 (Design Concept)
- **비주얼 테마**: Dark Neon / Cyberpunk
- **컬러 팔레트**: #000000 (Bg), #00ff99 (Primary), #ff0066 (Accent), #ffff00 (Coin)
- **폰트**: Orbitron (타이틀), Inter/Noto Sans KR (본문)
- **애니메이션 원칙**:
    - MMOC 변동 시 숫자 롤링 애니메이션
    - 등급 승급 시 전체화면 축하 이펙트
    - 버튼 호버/클릭 시 네온 글로우 효과
    - 페이지 전환 시 부드러운 슬라이드 트랜지션

## 9. 시스템 아키텍처 (System Architecture)
### 9.1. 기술 스택 (Tech Stack)
- **Frontend**: React (Vite), TypeScript, Tailwind CSS
- **Backend**: Node.js, Express
- **Real-time Communication**: **Socket.IO** (Room 기반 게임 세션 관리)
- **Blockchain**: **MetaMask + Web3.js/ethers.js** (MOC 토큰 입출금, ERC-20 연동)
- **인증**: MetaMask 지갑 서명 기반 인증 (Web3 Auth)
    - **Fallback**: MetaMask 미설치 시 이메일/소셜 로그인 후 내부 지갑 자동 생성 (게스트 모드, 프리 모드만 가능)
    - **지갑 연결 후 업그레이드**: 게스트 → MetaMask 연결 시 기존 EXP/전적 유지, 입출금 기능 해금
- **Database**:
    - **PostgreSQL/MySQL**: 사용자 정보, 자산(MMOC), 입출금 로그 (ACID 트랜잭션 보장)
    - **Redis**: 실시간 게임 세션 상태, 채팅 캐싱, 리더보드 (고속 Read/Write)

### 9.2. 서버 구조 (Server Structure)
- **API Server**: 로그인, 랭킹 조회, 마켓플레이스 등 (RESTful)
- **Game Server**: Socket.IO 기반 실시간 게임 로직 처리 (Server-Authoritative)
    - **Room Manager**: 게임 방 생성/삭제, 매칭 관리
    - **Game Loop**: 약 20~60 FPS로 게임 상태 연산 및 브로드캐스팅
    - **Matchmaker**: Redis Queue 기반 매칭 대기열 관리

## 10. 데이터베이스 설계 (Database Schema)
### 10.1. Users Table
| 컬럼 | 타입 | 설명 |
|------|------|------|
| `uuid` (PK) | UUID | 사용자 고유 ID |

## 11. 경제 밸런스 (Economy Balance)

### 핵심 경제 구조: MOC ↔ MMOC 듀얼 토큰 + PvP 제로섬
> 유저는 메타마스크 지갑의 **MOC(외부 토큰)**을 **입금소**를 통해 **MMOC(인게임 포인트)**로 전환합니다.
> MMOC로 게임을 플레이하고, 획득한 MMOC는 다시 MOC로 **출금** 가능합니다.
> 소액 유저의 재방문을 위해 **Faucet(무료 충전소)**를 제공합니다.

### 입/출금 흐름
```
[메타마스크 지갑] MOC 보유
        │
    [입금소] MOC → MMOC (1:1 전환)
        │
    [아케이드 인게임] MMOC로 플레이
        │
    [출금소] MMOC → MOC (1:1 전환, 수수료 2%)
        │
[메타마스크 지갑] MOC 획득
```

### PvP 배팅 흐름 (1v1 예시)
```
[Player A] 100 MMOC 배팅 ──┐
                          ├── 총 판돈: 200 MMOC
[Player B] 100 MMOC 배팅 ──┘
                 │
           게임 진행 (대결)
                 │
        ┌────────┴────────┐
    [A 승리]           [B 승리]
   +200 MMOC          +200 MMOC
```

### 다인전 배팅 흐름 (8인 Neon Deathmatch 예시)
```
8명 × 50 MMOC = 총 판돈 400 MMOC
    │
  게임 진행 (배틀로얄)
    │
  최종 순위 (분배형):
    1등 → 240 MMOC (60%)
    2등 → 100 MMOC (25%)
    3등 →  60 MMOC (15%)
    4~8등 → 배팅금 전액 소멸
```

### MMOC 유입 (Sources)
| 항목 | 설명 |
|------|------|
| **MOC 입금** | 메타마스크 지갑에서 MOC → MMOC 1:1 전환 (핵심 유입 경로) |
| **PvP 승리** | 상대방이 건 판돈 획득 (MMOC 이동, 신규 생성 아님) |
| **Faucet (무료 충전)** | 잔고 1,000 MMOC 미만 시 봇 인증 후 1,000 MMOC 지급 (일 3회 제한) |

### MMOC 유출 (Sinks / Drains)
| 항목 | 설명 |
|------|------|
| **PvP 패배** | 핵심 Sink – 패자의 판돈이 승자에게 이동 |
| **출금 수수료** | MMOC → MOC 출금 시 2% 수수료 (Sink) |
| **토너먼트 참가비** | 참가비의 10% = 운영 수수료 |
| **UGC 마켓플레이스** | 스킨 구매 시 MMOC 소멸 (판매자 85%, 수수료 15%) |
| **기프티콘 교환** | MMOC로 실제 기프티콘 구매 (영구 소멸) |

### 경제 안정 정책
- **제한적 Faucet**: 잔고 1,000 미만 + 일 3회로 무료 충전 제한 → 인플레이션 최소화
- PvP 구조 자체가 **제로섬**: MMOC가 유저 간 이동할 뿐, 시스템에서 새로 생성되지 않음
- 출금 수수료(2%) + 토너먼트(10%) + UGC(15%) = 다중 Sink
- Faucet 어빌징 방지: 봇 인증(캡차) + 동일 지갑 제한 + 일일 횟수 제한

## 12. 보안 및 어뷰징 방지 (Security)
- **Server-Authoritative**: 모든 피격, 이동, 점수 계산은 서버에서 검증 (클라이언트는 렌더링만 담당)
- **Rate Limiting**: Socket 이벤트 도배 방지
- **트랜잭션 검증**: 클라이언트가 보낸 배팅 금액이 실제 서버 보유량과 일치하는지 더블 체크
- **에스크로(Escrow) 방식**: 배팅 시 양쪽 코인을 서버가 **선차감 후 보관** → 게임 종료 후 승자에게 지급 (중간 이탈 방지)
- **중복 로그인 방지**: 동일 계정 동시 접속 차단 (Socket Session ID 관리)
- **다계정 감지**: 동일 IP/디바이스 핑거프린트에서 다수 계정 생성 시 경고
- **승부조작 감지**: 동일 유저 간 반복 대결 + 한쪽만 승리하는 패턴 → 자동 경고 및 로깅
- **비정상 행위 감지**: 연속 즉시 배팅+탈퇴 패턴, 매크로 의심 행위 로깅

## 13. 운영 및 관리 도구 (Admin & Operations)
### 13.1. Admin Dashboard
- **유저 관리**: 유저 검색, MMOC 수정(보상/회수), 밴(Ban) 처리, 지갑 동결
- **게임 로그 조회**: 특정 유저의 배팅/승패 기록 원장 조회 (분쟁 해결용)
- **PvP 매칭 로그**: 대전 상대, 판돈, 결과를 한눈에 확인 (승부조작 감지)
- **공지사항 관리**: 로비 상단 롤링 공지 등록
- **실시간 모니터링**: 현재 접속자 수(CCU), 활성화된 게임 방 현황
- **경제 대시보드**: 전체 MMOC 유통량, MOC 입/출금 현황, 일일 Faucet 사용량, 인플레이션 모니터링

### 13.2. 재접속 및 네트워크 처리 (Network Stability)
- **재접속(Reconnection)**:
    - 유저가 게임 중 새로고침/네트워크 끊김 발생 시 `Socket ID`가 끊기더라도 `User ID` 기준으로 30초 내 재접속 시 기존 방/상태 유지
- **비정상 종료 처리**:
    - 배팅 후 게임 강제 종료 시 '패배(기권)' 처리 원칙 (악의적 판돈 회피 방지)
    - 서버 다운 등 불가항력 시 해당 판 무효화 및 배팅금 롤백 (에스크로에서 양측 환불)
